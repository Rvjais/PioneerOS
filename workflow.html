Authentication Flow

Magic Link Flow

  1. User enters email/phone/empID
  2. API generates 30-min JWT token
  3. Token sent via WhatsApp/Email
  4. User clicks link → validates token
  5. NextAuth session created

Password Flow

  1. User enters credentials
  2. Server validates bcrypt hash
  3. Creates secure session cookie
  4. Redirects to dashboard
  5. Role detected from DB

Role Assignment

  1. Fetch user from Prisma
  2. Check role & department
  3. UnifiedSidebar selects nav
  4. 11 department-specific navs
  5. RBAC applied to all APIs
